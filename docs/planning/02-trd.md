## 2. 권장 기술 스택

### 결정 방식
- **사용자 레벨**: L2
- **결정 방식**: 사용자 직접 선택

### 2.1 프론트엔드 + 백엔드

| 항목 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | Ruby on Rails | 사용자 선택 — 풀스택 프레임워크로 빠른 MVP 개발 가능 |

### 2.2 AI 서버

| 항목 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | FastAPI (Python) | AI/ML 라이브러리 생태계 활용, 비동기 처리 지원 |

### 2.3 데이터베이스

| 항목 | 선택 | 이유 |
|------|------|------|
| RDBMS | PostgreSQL | 안정성, 확장성, 무료 |
| 벡터 검색 | pgvector | 검증된 팩트 임베딩 저장 및 유사 검색 |

### 2.4 인증

| 항목 | 선택 | 이유 |
|------|------|------|
| 인증 시스템 | Authentication Zero + Email OTP | 보안성, 간편한 이메일 인증 |

### 2.5 영상 분석 파이프라인

| 단계 | 도구 | 역할 |
|------|------|------|
| 1 | yt-dlp | 유튜브 영상/자막 다운로드 |
| 2 | Whisper | 음성 → 텍스트 변환 (자막 없는 영상 대응) |
| 3 | OpenAI API | 텍스트에서 팩트 주장 추출 |
| 4 | 빅카인즈 뉴스 API | 추출된 주장과 관련 뉴스 데이터 대조 |
| 5 | OpenAI API | 대조 결과 기반 검증 리포트 생성 |

### 2.6 리포트 생성

| 형식 | 도구 |
|------|------|
| PDF | WeasyPrint 또는 ReportLab |
| DOCX | python-docx |
| HWP | pyhwp 또는 hwplib |
| MD | 기본 텍스트 생성 |

## 3. Decision Log

| 결정 | 대안 | 선택 이유 |
|------|------|----------|
| Ruby on Rails | Next.js, Django | 사용자 선택 — 풀스택 단일 프레임워크로 빠른 개발 |
| FastAPI | Flask, Django REST | Python AI 생태계 + 비동기 + 자동 API 문서화 |
| PostgreSQL + pgvector | MongoDB, Pinecone | 단일 DB로 관계형 데이터 + 벡터 검색 모두 처리 |
| Authentication Zero | Auth0, Firebase Auth | 사용자 선택 |
| yt-dlp + Whisper | YouTube API 자막 | 자막 없는 영상도 처리 가능 |

## 4. 비기능 요구사항

| 항목 | 기준 |
|------|------|
| 팩트체크 분석 시간 | 90초 이내 (10분 이내 영상 기준) |
| 동시 분석 요청 | 최소 10건 |
| 가용성 | 99.5% |
| 데이터 보존 | 리포트 최소 1년 보존 |
| 보안 | HTTPS 필수, 인증 토큰 기반 API 통신 |
