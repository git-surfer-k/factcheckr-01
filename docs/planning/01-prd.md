# PRD (Product Requirements Document)

> Factis — 유튜브 시사/뉴스 채널 AI 팩트체크 플랫폼

---

## MVP 캡슐

1. **목표**: 유튜브 시사/뉴스 채널의 주장을 빅카인즈 뉴스 빅데이터로 AI 팩트체크하고, 채널 신뢰도를 평가하며, 기업에 광고적합성 리포트를 제공하는 플랫폼
2. **페르소나**: 시사 뉴스를 즐겨보는 시민 — "이 채널 계속 봐도 되나?" 의문이 드는 순간 사용
3. **핵심 기능**: FEAT-1 AI 팩트체크 엔진, FEAT-2 채널 신뢰도 평가, FEAT-3 B2B 광고적합성 리포트
4. **성공 지표 (노스스타)**: 월간 팩트체크 리포트 생성 수
5. **입력 지표**: 일일 URL 제출 수, 리포트 다운로드 전환율
6. **비기능 요구**: 팩트체크 분석 결과 90초 이내 반환
7. **Out-of-scope**: 크롬 확장 프로그램, 실시간 스트리밍 분석, 모바일 네이티브 앱
8. **Top 리스크**: AI 팩트체크 정확도 — 환각(hallucination)으로 인한 오검증
9. **완화/실험**: 뉴스 근거 링크 필수 첨부 + 신뢰도 점수에 "AI 검증 한계" 표시
10. **다음 단계**: /screen-spec으로 화면별 상세 명세 작성

---

## 1. 배경 및 문제 정의

### 1.1 문제

- 유튜브 시사/뉴스 채널에서 검증되지 않은 주장이 사실처럼 전파됨
- 시청자는 해당 주장의 사실 여부를 직접 확인할 수단이 없음
- 기업은 광고 집행 시 채널의 브랜드 안전성을 판단하기 어려움

### 1.2 기회

- 빅카인즈가 보유한 1.1억 건 뉴스 빅데이터와 AI를 결합하면 자동 팩트체크가 가능
- 채널 신뢰도를 정량화하여 시청자와 광고주 모두에게 가치를 제공

---

## 2. 타겟 사용자

### 2.1 B2C — 시사 뉴스 시청자

| 항목 | 설명 |
|------|------|
| 누구 | 유튜브로 시사/뉴스를 즐겨보는 시민 |
| 사용 계기 | "이 채널 계속 봐도 되나?" 의문이 드는 순간 |
| 핵심 니즈 | 채널/영상의 주장이 사실인지 빠르게 확인 |
| 빈도 | 주 2~3회 |

### 2.2 B2B — 광고주/광고대행사

| 항목 | 설명 |
|------|------|
| 누구 | 유튜브 광고 집행을 담당하는 기업/대행사 |
| 사용 계기 | 광고 집행 전 채널 브랜드 안전성 평가 |
| 핵심 니즈 | 채널별 광고적합성 심층 리포트 |
| 빈도 | 캠페인 기획 시 (월 1~2회) |

---

## 3. 핵심 기능

### FEAT-1: AI 팩트체크 엔진

| 항목 | 설명 |
|------|------|
| 설명 | 유튜브 영상 URL 입력 → AI가 자막 분석 → 팩트 주장 추출 → 빅카인즈 뉴스 데이터 대조 → 종합 검증 리포트 생성 |
| 사용자 가치 | 클릭 한 번으로 영상의 사실 여부 확인 |
| 분석 파이프라인 | yt-dlp → Whisper → OpenAI API(주장추출) → 빅카인즈 뉴스 API(뉴스대조) → OpenAI API(검증리포트) |

### FEAT-2: 채널 신뢰도 평가 시스템

| 항목 | 설명 |
|------|------|
| 설명 | 팩트체크 정확도, 출처 인용률, 논조 일관성 등 다차원 지표로 채널 평가 + 시계열 추적 + 스코어보드 랭킹 |
| 사용자 가치 | 채널을 계속 볼지 말지 판단할 수 있는 객관적 지표 |
| 카테고리 | 정치 / 경제 / 사회 / 국제 + 사용자 지정 태그 |

### FEAT-3: B2B 광고적합성 리포트

| 항목 | 설명 |
|------|------|
| 설명 | 기업/상품 정보 입력 → AI가 추천 채널 리스트 + 광고적합성 심층 리포트 자동 생성 |
| 사용자 가치 | 광고 집행 전 채널의 브랜드 안전도를 정량적으로 평가 |
| 수익 모델 | 월 정기구독 |

## 5. 벤치마킹 (유사 서비스 분석)

| 서비스 | 특징 | Factis 차별점 |
|--------|------|--------------|

## 6. 가설 vs 사실

| # | 내용 | 구분 | 검증 계획 |
|---|------|------|----------|
| 1 | 시청자가 유튜브 영상의 사실 여부에 관심이 있다 | 가설 | MVP 출시 후 URL 제출 빈도 측정 |
| 2 | 빅카인즈 뉴스 데이터로 유튜브 주장을 검증할 수 있다 | 가설 | 100개 영상 파일럿 테스트로 검증 정확도 측정 |
| 3 | 기업이 채널 광고적합성 리포트에 비용을 지불할 의향이 있다 | 가설 | 광고대행사 5곳 인터뷰 |
| 4 | 빅카인즈 API로 1.1억 건 뉴스 데이터에 접근 가능하다 | 사실 | 빅카인즈 API 문서 확인 완료 |

## 7. 핵심 가치 및 타협

### 핵심 가치 (타협 불가)

- **검증 투명성**: 모든 팩트체크 결과에 근거 뉴스 링크 필수 첨부
- **객관성**: AI 분석 결과에 "AI 검증 한계" 명시, 점수 산출 근거 공개

### 타협 사항 (의도적 생략)

| 타협 | 이유 |
|------|------|
| 크롬 확장 프로그램 | MVP에서는 웹앱에 집중, 이후 확장 |
| 실시간 스트리밍 분석 | 기술적 복잡도 높음, 녹화 영상부터 시작 |
| 사용자 팩트체크 참여 (크라우드소싱) | 초기에는 AI 자동화에 집중 |

## 8. 실험 및 실패 대응

### MVP 실험

| 항목 | 내용 |
|------|------|
| MVP 범위 | FEAT-1(팩트체크) + FEAT-2(채널 신뢰도) + FEAT-3(B2B 리포트) |
| 성공 기준 | 출시 3개월 내 월간 리포트 1,000건 생성 |
| 실패 기준 | 3개월 내 월간 리포트 100건 미만 |

### 실패 시나리오 및 학습 목표

| 시나리오 | 학습 목표 | 대응 |
|----------|----------|------|
| AI 팩트체크 정확도 낮음 | 어떤 유형의 주장에서 오류 발생하는지 파악 | 주장 유형별 검증 파이프라인 세분화 |
| 사용자 유입 저조 | 타겟 사용자의 진입 경로 분석 | SNS 카드뉴스 자동생성 등 바이럴 기능 추가 |
| B2B 수요 부재 | 기업이 원하는 리포트 형태 재조사 | 리포트 형식/내용 피벗 |
